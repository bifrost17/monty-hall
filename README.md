# 🚪 몬티 홀 문제 (Monty Hall Problem)

갤럭시 폴드 펼친 화면 기준으로 최적화한 인터랙티브 몬티 홀 문제 웹페이지.
문 갯수를 직접 정하고, 호스트가 차 위치를 지정한 뒤 참여자가 게임을 진행합니다.

## 파일

| 파일 | 설명 |
|------|------|
| [`index.html`](index.html) | 기본 버전 — 단계별 위저드 진행 |
| [`animated.html`](animated.html) | 애니메이션 버전 — 문 갯수 확정 후 **문 화면 고정**, 문 여닫힘 3D 애니메이션 + 당첨/꽝 이펙트(색종이·흔들림) |

## 진행 순서

1. **문의 갯수 입력** (최소 3개)
2. **문 생성 + 차 위치 지정** — 호스트가 🚗 위치 지정 (이후 가려짐)
3. **참여자 선택** — 문 하나 선택
4. **몬티가 제거** — 당나귀 문을 열어 닫힌 문 2개만 남김
5. **파이널 선택** — 그대로 두기 / 바꾸기
6. **최종 정답 확인** — 당첨/꽝 + 이론 확률 + 누적 전적

## 핵심 결론

문이 N개일 때 처음 선택이 정답일 확률은 `1/N`, **바꾸면 `(N-1)/N`** 로 항상 유리합니다.
